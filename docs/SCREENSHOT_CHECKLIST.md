# 截图清单与拍摄规范

本文用于整理公开仓库文档所需的截图位，先定义拍摄范围和规范，后续再补正式图片。

## 1. 目标

- 为 `README.md` 和 `docs/USER_GUIDE.md` 提供统一的截图规划
- 保证未来补图时风格一致、信息脱敏、结构清晰
- 避免在公开仓库中暴露真实项目路径、账号信息和私有数据

## 2. 总体规范

### 脱敏要求

截图中不要出现以下内容：

- 真实项目名
- 真实用户目录
- 真实公司 / 产品 / 角色命名
- API Key
- 私有仓库路径
- 内网路径

建议统一替换为示例路径，例如：

- `D:\Captures\sample_before.rdc`
- `D:\Captures\sample_after.rdc`
- `D:\Projects\SampleUEProject`

### 视觉规范

- 使用同一套窗口尺寸
- 保持深色主题的一致视觉效果
- 避免同一张图中信息过多、字体过小
- 优先选择“刚完成一次操作、结果已出现”的状态

### 推荐窗口规格

- 宽度：`1600`
- 高度：`980`
- 最小不要低于应用默认窗口的可视范围

## 3. README 截图清单

README 只建议放少量总览图，避免首页过长。

### 图 1：应用首页总览

- 目标：让读者一眼看到工具是多功能桌面工作台
- 推荐内容：
  - 顶部标题
  - 四个 Tab
  - 三栏布局中的主要区域
- 推荐来源：
  - 默认进入应用后
  - 或切到信息密度最高的一页拍总览图

### 图 2：性能 Diff 报告

- 目标：展示工具不仅有表单，还能直接查看报告结果
- 推荐内容：
  - 左侧表单
  - 中间 HTML 报告区域
  - 右侧历史 / 日志

### 图 3：资产批量导出或性能分析

- 二选一，取决于你更希望公开仓库突出哪一类能力
- 若强调资源提取流程：拍 `资产批量导出`
- 若强调定位性能问题：拍 `性能`

## 4. USER_GUIDE 截图清单

`USER_GUIDE.md` 可以放更多分功能截图。

### A. 环境设置

- 界面区域：
  - `环境设置` 弹窗
  - 检测结果区
- 目标：
  - 说明首次启动如何完成配置

### B. 问题诊断

- 界面区域：
  - Before / After 输入区
  - 问题描述
  - 结果报告区
- 目标：
  - 展示“输入抓帧 + 问题描述 -> 输出报告”的主流程

### C. 性能 Diff

- 界面区域：
  - Base / New 输入区
  - 报告 iframe
  - 运行日志
- 目标：
  - 展示对比分析的完整结果页

### D. 单帧性能

- 界面区域：
  - 排序工具条
  - 性能表格
  - 右侧图表
- 目标：
  - 展示热点排序与分布信息

### E. 资产批量导出

- 界面区域：
  - Pass 读取与范围选择
  - 导出格式选项
  - 日志或导出清单
- 目标：
  - 展示导出工作流入口

### F. CSV 列映射预览

- 界面区域：
  - 多行路径输入区
  - `浏览` / `目录`
  - 映射下拉项
  - 转换按钮
- 目标：
  - 展示 CSV 到模型转换的实际使用方式

### G. 批量映射确认

- 界面区域：
  - 批量映射确认弹窗
  - 关键映射选择项
- 目标：
  - 展示导出前确认映射这一差异化功能

## 5. 推荐拍摄顺序

1. 先准备一套统一的演示样本路径
2. 先拍 README 用的 1-3 张总览图
3. 再拍 `USER_GUIDE.md` 的分功能图
4. 最后统一裁切、命名并补到文档中

## 6. 图片命名建议

建议统一放在未来的文档图片目录，例如：

```text
docs/images/
  overview-home.png
  diagnose-report.png
  cmp-report.png
  perf-analysis.png
  asset-export.png
  csv-mapping.png
  mapping-confirm-modal.png
  setup-modal.png
```

## 7. 后续补图时的检查项

- 是否出现真实路径
- 是否出现真实账号信息
- 是否出现私有项目名
- 是否出现真实 API Key
- 是否有过多无关空白
- 是否能一眼看出当前截图要表达的功能
