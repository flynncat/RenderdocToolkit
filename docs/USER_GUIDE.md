# 用户使用指南

本文面向第一次接触本项目的用户，介绍如何使用桌面工具中的四个主要功能页。

## 1. 界面概览

应用启动后，顶部会显示四个主要 Tab：

- `问题诊断`
- `性能 Diff`
- `性能`
- `资产批量导出`

右上角提供两个全局入口：

- `环境设置`
- `刷新环境状态`

建议首次启动时先打开 `环境设置`，确认 RenderDoc 与模型服务相关配置可用。

![应用首页总览](images/overview-home.png)

## 2. 环境设置

`环境设置` 弹窗主要用于确认以下项目：

- `RenderDoc Python Path`
- `LLM Provider`
- `OpenAI-compatible Base URL`
- `OpenAI-compatible API Key`
- `OpenAI-compatible Model`
- `renderdoc_cmp 根目录`

建议检查流程：

1. 确认 RenderDoc 已安装
2. 填写或校正 RenderDoc Python 路径
3. 如果要启用在线模型，填写 OpenAI-compatible 配置
4. 确认 `renderdoc_cmp` 路径正确
5. 点击 `保存并应用`
6. 观察下方检测结果，确认工具链已准备完成

## 3. 问题诊断

适用于比较两份抓帧之间的表现差异，并结合问题描述生成诊断报告。

### 输入项

- `Before .rdc 路径`
- `After .rdc 路径`
- `目标 Pass`
- `问题描述`
- 可选：`EID Before`
- 可选：`EID After`

### 推荐使用方式

1. 选择两份抓帧
2. 输入你关心的 Pass 名称
3. 用简洁语言描述现象，例如“面部发亮，身体正常”
4. 如果已知事件号，可补充 Before / After 对应 EID
5. 点击 `开始分析`

### 输出区域

- `当前 Session`
- `首轮报告`
- `EID 深挖报告`
- `UE 源码扫描报告`
- `Session 与追问`

### 常见工作流

1. 先用首轮报告看整体差异
2. 如果已定位到具体 draw，再执行 `EID 深挖`
3. 如果问题与引擎逻辑相关，再执行 `UE 源码扫描`
4. 最后在追问框里继续提出更细的问题

## 4. 性能 Diff

适用于对比两份 `.rdc` 的性能表现差异。

### 输入项

- `Base .rdc 路径`
- `New .rdc 路径`
- 可选：`RenderDoc 目录`
- 可选：`Malioc 路径`
- 可选：`Strict Mode`
- 可选：`Verbose`

### 使用步骤

1. 选择两份待比较的抓帧
2. 如有需要，补充 RenderDoc 或 Malioc 路径
3. 点击 `执行性能 Diff`
4. 查看中间区域生成的报告
5. 结合右侧历史与日志定位异常变化

### 输出区域

- `cmp 结果`
- HTML 报告内嵌窗口
- `cmp 历史`
- `运行日志`

![性能 Diff 页面](images/cmp-report.png)

## 5. 性能

适用于分析单个 `.rdc` 的 Pass 级性能热点。

### 输入项

- `RenderDoc .rdc 路径`

### 使用步骤

1. 选择单个抓帧
2. 点击 `执行性能分析`
3. 使用排序维度和排序方向调整结果顺序
4. 查看性能表中的热点项
5. 结合右侧图表和提示信息理解开销分布

### 可关注信息

- 排序后的热点 Pass
- 图表中的阶段开销分布
- 线框预览
- 运行日志中的附加提示

## 6. 资产批量导出

适用于从抓帧中批量导出资产，并对 CSV 做模型格式转换。

### 主要流程

1. 读取 Pass 列表
2. 选择导出范围
3. 设定输出格式
4. 在导出前确认批量映射
5. 查看导出结果与日志

### Pass 选择

支持以下方式：

- 单个 Pass
- Pass 区间
- 手动输入单个 EID
- 手动输入起始 / 结束 EID

### 导出格式

- `FBX`
- `OBJ`
- 贴图格式：`PNG`、`TGA`、`HDR`、`EXR`

## 7. CSV 列映射预览

该区域既可作为资产导出的辅助，也可以独立使用。

### 支持的输入方式

- 单个 CSV 文件
- 多个散点 CSV 文件路径
- 单个目录

如果输入多个文件路径，建议每行一个路径。

### 推荐步骤

1. 通过 `浏览` 选择一个或多个 CSV 文件
2. 或通过 `目录` 选择一个包含 CSV 的目录
3. 点击 `识别列映射`
4. 检查 `Position`、`Normal`、`UV`、`Tangent` 等列是否合理
5. 如有需要手动调整映射
6. 点击 `按当前映射转换 CSV/目录`

### 当前行为

- 如果当前已经选中某个资产导出任务，则转换结果会挂到该任务下
- 如果当前没有选中任务，工具会自动创建一个“手工 CSV 转换任务”

![资产批量导出与 CSV 映射](images/asset-export.png)

## 8. 常见问题

### Q1：为什么界面提示某些环境未就绪？

通常是以下路径未配置正确：

- RenderDoc Python Path
- `renderdoc_cmp` 根目录
- 在线模型配置

建议优先到 `环境设置` 中核对。

### Q2：CSV 转换必须先做资产批量导出吗？

不必须。当前 CSV 转换支持独立使用；如果没有选中导出任务，工具会自动创建一个手工转换任务。

### Q3：为什么不同抓帧来源下信息显示不完全一致？

不同 API、不同平台和不同抓帧方式下，RenderDoc 可提供的数据粒度不同，移动端或真机抓帧尤其需要结合原始 RenderDoc 视图一起判断。
